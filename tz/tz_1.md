**Задание для квалификационного экзамена**

Реализовать бэкенд веб-приложения из технического задания. Упаковать проект в docker-compose. Требования к контейнерам:

- запуск контейнера осуществляется не из под root;
- в случае использования компилируемого языка образ не должен содержать файлов с исходным кодом;
- healthcheck реализован на все контейнеры (в том числе web);
- требуемые контейнеры в порядке запуска:
  - база данных (например, postgres);
  - web (сам сервер с исходным кодов);
  - nginx (как маршрутизатор для web);
- реализован volume для связи между локальным хранилищем и кодом в контейнере;
- volume для сохранения данных используемой базы данных;
- открытый порт из docker-compose сети может быть только 80 у контейнера nginx.

Подготовить декоратор или иной инструмент для подсчета Latency интеграционных запросов через Histogram (например, в базу данных).

Реализовать Middleware для подсчета количества запросов на сервис. Требуемые счетчики - общее количество запросов, 2хх, 4хх и 5хх статусы ответов.

Реализовать отдельный конфигурационный модуль для получения настроек из переменных окружения.

Реализовать Middleware, которая сохраняет или создает Correlation-ID для последующего использования в логере. Вывод лог сообщения в виде: «[<Correlation-ID из запроса>] <log message>».

Реализовать интеграционный тест с вызовом пустого endpoint и проверкой на 200-ый код ответа.

Установить и настроить линтер для выбранного языка программирования(для python — ruff).
